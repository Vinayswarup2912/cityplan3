Course Overview - Algorithms

1. Problems in Nature

Iteration
  • Repeats steps in a loop until the job is done.
  • Examples: The sun rising and setting, counting tree rings.
Recursion
  • Solves a problem by breaking it into smaller steps.
  • Examples: Tree branches, factorials (5! = 5 × 4 × 3 × 2 × 1).
Backtracking
  • Uses trial and error to find solutions.
  • Examples: Ants finding food, solving mazes.
Divide and Conquer
  • Merging Records: Breaks and combines data.
  • Sorting: Splits, sorts, and merges data.
Greedy Algorithm
  • ATM Withdrawal: Takes the biggest bills first.
  • Activity Scheduling: Chooses the task that ends quickest.
Dynamic Programming
  • Route Planning: Solves parts of a route to find the best one.
  • Fibonacci: Remembers results to avoid doing the same work again.
Graph Algorithms
  • Dijkstra (Shortest Path): Finds the quickest route.
  • Prim's/Kruskal's (MST): Connects with the least cost.
Sorting Algorithms
  • Task Sorting: Organizes tasks to get them done efficiently.
  • Ticket Sorting: Sorts tickets by price.
Searching Algorithms
  • Linear Search: Finds a file.
  • Binary Search: Finds a word in a sorted list.

2. Space and Time Efficiency

Space Efficiency

How much memory an algorithm uses. Fewer resources are better.

Time Efficiency

How fast an algorithm runs. Faster performance leads to better outcomes.

Order Time Space
O(1) Array access Single variable
O(log n) Binary search Recursive stack
O(n) Loop through list Storing list
O(n log n) Merge Sort Auxiliary space
O(n²) Bubble Sort 2D matrix
O(2^n) Brute force Storing subsets
O(n!) Generating permutations Storing permutations
Why It Matters
  • Optimized Performance: Faster processing
  • Efficient Memory Use: Saves memory
  • Scalability: Handles larger data sets
  • Real-World Impact: Improves system reliability
  • Resource Constraints: Smooth operation on limited devices

3. Design Principles

Shortest Path Trees
  • Finds the most efficient routes.
  • Essential for optimization and navigation challenges.
  • Real-world example: GPS routing.
Partitioning
  • Breaks down complex problems into smaller, manageable segments.
  • Reduces algorithmic complexity.
  • Example: Demonstrated in algorithms like quicksort.
Balancing and Rotations
  • Avoids performance degradation in search and update operations.
  • Examples: AVL trees, Red-Black trees.
Edge Relaxation
  • Updates shortest paths gradually during graph traversal.
  • Example: Prominent in Dijkstra's shortest path algorithm.
Level Order Traversal
  • Systematic, layer-by-layer problem exploration.
  • Method: Implements a breadth-first approach.
Pruning
  • Eliminates unnecessary or invalid solution paths.
  • Reduces computational complexity and saves time.
Brave and Cautious Travel Strategies
  • DFS: Enables deep, aggressive exploration.
  • BFS: Provides methodical, level-by-level investigation.
